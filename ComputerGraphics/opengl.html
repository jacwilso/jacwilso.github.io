Using the OpenGL library, I progammed applications in 2D and 3D, vertex and fragment shaders, bezier curves, and collision detection. The course emphasized the rendering pipeline and to understand the mathematics and operation of the stages in the pipeline.

These samples use the OpenGL library to animate, texture, and introduce a flat shader. The texture objects are wrapped using the SOIL library from .jpg images. A virtual class was written to texture cubic objects which allowed code reuse for the head of the right character and the skybox. The center object was import from an .obj file and a minimum-pass through shader was applied. Although an introductory course, the fixed-function pipeline and flat shaders wer eventually replaced with shaders reimplementing the Blinn-Phong model. The characters are bound to the grid, thus at the boundary conditions it will stop advancing.

Bezier curves and surfaces, viewports, object manipulation, parametric arc length and arc step animation, and various camera types are used in the other project. The character is bound to the Bezier surface while he moves throughout the scene. He rotates with the normals of the curve, as well as stopping at the edges. The user can chose first person, third person, sky view, and free camera. Another two models are travelling along the bezier curve in the background with the different animations and the camera can be switched to their perspective. A basic collision detection, complexity of O(n^2), was used to detect collision with the objects placed on the scene.

For my end semester project my team of 3 wanted to implement a realistic solar system, so we used bump map shaders to provide visual contrast of the planets rather than plain texturing. The relative size of the system was also compromised, and the bump maps provided better performance over realistic meshes. Since then, I have continued expanding my knowledge with computer graphics and shaders through text, such as Real-Time Rendering, and Unity shader programming.

2D graphics focus on the basics of the rendering pipeline, verticies to primitives to color on the screen. With this strict focus, I developed a strong basis which was able to be transferred into 3D.

The sample character was designed to replicate Wilson from Don't Starve and the background to replicate Pallet Town from Pokemon. The user can control the character about the screen and at the edges the character will transistion to a new background image. The character is drawn hierarchically to animate movement. The user can click about the screen to draw and animate a torch at the mouse location.

The game sample is an individual project which emphasizes pixel art and simple collision detection. Each block was recreated by hand in OpenGL by placing colored quads. A text file was used to describe the level layout (each model had a distinct character), thus a different text file loaded a different level. Enemies are placed throughout the scene and are animated similar to the game to obstruct the player. The collision detection handles the character jumping, landing on ground or enemies, gravity, and collision with blocks above. The collision detection used a grid system to check movement and enemies, however this caused choppy gameplay. To remedy the gameplay issue, later on in the semester, I implemented the primitive collision detection at O(n$^2$) cost.

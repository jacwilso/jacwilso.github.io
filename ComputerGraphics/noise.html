Noise Visualization

Language: C# (Unity)

Contributions
  • Programmed value and Perlin noise with 2D and 3D functions
  • Applied pseudorandom noise onto grayscale and gradient textures and 3D mesh

I programmed random and pseudorandom noise functions which would generate texture gradients, for the purpose of implementing them into a procedurally generating terrain.

To begin, I programmed a script to allow pixels to be shaded at variable resolutions on a texture. Adding in simple noise by randomly varying the color white created a TV fuzz looking texture; however this was undesirable because there was harsh transisitions between adjacent pixels. I used lattice noise and a hashing array to produce the pseudorandom pattern, and restricted the resolution to 256 based on the size of my hash array. For the 2D noise implementation, I started by programming noise in 1D (vertically) then transitioned into 2D noise function. To resolve the sharp transition between adjacent pixels, I implemented Value and Perlin noise. I programmed the noise function to handle fractal noise to be able to add additional details lesser pronounced than the larger features. I implemented an editor gui which controlled the dimensionality of the noise, frequency, resolution, noise type, octaves, and gradient coloring. To transition into 3D, rather than using a quad and I pixels, I used vertex vertical positioning. I programmed my own generated mesh, which allowed the scalable resolution from the 2D framework. I calculated the normals of the mesh after the alterations to be able to handle incoming light by programming the finite difference formula in multiple dimensions.
